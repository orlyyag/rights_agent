# ⁧עוזר הזכויות של "כל זכות" — תקציר למנחה⁩

⁧**מה זה:** בוט Telegram דו־לשוני (עברית + רוסית) שעונה על שאלות זכויות בישראל — "מה מגיע לי אחרי לידה?" — מקורקע בדפי "כל זכות" אמיתיים, עם עד 3 ציטוטים, סירוב מנומס כשהשאלה מחוץ לתחום, ודיסקליימר משפטי. ⁦~3⁩ שניות לתשובה, ⁦$0.010⁩ לשאלה.⁩

## ⁧הארכיטקטורה — שני מישורים שלא נפגשים⁩

⁧**המסלול החי (~3 שניות):**⁩

```
Telegram → Guardrails → Retriever (Chroma, top-8) → Gemini 3.5 Flash → HTML + citations → Telegram
```

⁧**המסלול האופליין (פייפליין ingestion שבנינו בעצמנו):**⁩

```
MediaWiki API → acquire (manifest-diff) → clean (tables→Markdown) → chunk (~512 tok) → embed (3072 dim) → Chroma
```

- ⁧**הרעיון המרכזי: active pointer בלו־גרין.** האינדקס החדש נבנה בצד, והמצביע מתחלף אטומית — אף שאילתה חיה לא פוגשת אינדקס חצי־בנוי. גם rollback הוא שורה אחת.⁩
- ⁧**מסלול אג'נטי ב-LangGraph (opt-in):** rewrite → retrieve → grade_docs → re-retrieve פעם אחת → generate/refuse. ה-grade הוא קריאת LLM אחת batched על כל המועמדים — לא קריאה לכל chunk.⁩

## ⁧ההחלטות המעניינות (עם ה"למה")⁩

1. ⁧**טבלאות → Markdown זה הניצחון המדיד הכי גדול.** הקורפוס המוכן של Webiks היה snapshot ממאי 2024 עם מענק לידה של 1,986 ₪; הפייפליין שלנו מושך מהאתר 2,103 ₪ — הסכום הנכון להיום. רעננות ניצחה תחכום.⁩
2. ⁧**הלולאה האג'נטית נשארה opt-in — כי מדדנו.** הרצנו ⁦2×2⁩ (linear/agent × corpus/pipeline) על 10 מקרי כשל: הפייפליין החזיר 40% מהחמצות ה-retrieval, האייג'נט כמעט לא הוסיף מעליו. ברירת המחדל היא linear + pipeline — הכי מדויק והכי זול. לא שילמנו פי 4 בקריאות LLM על feature שלא הוכיח את עצמו.⁩
3. ⁧**שופט cross-provider:** הבוט 100% Gemini, אבל ה-eval נשפט ע"י מודל של OpenAI — כדי שלא יהיה self-preference של מודל שמצ'פר את התשובות של עצמו.⁩
4. ⁧**Frozen seams:** שלושה קבצים (config, llm wrapper, schema) ננעלו ביום הראשון כחוזה בין מסלולי הבנייה — ingestion ו-serving יכלו להתפתח במקביל בלי לשבור זה את זה.⁩
5. ⁧**Guardrails בכניסה וביציאה:** allowlist, מגבלת קצב, אורך מינימום/מקסימום, עמידות בהזרקת פרומפט, ו-PII redaction בלוגים. 100% סירוב נכון על סט adversarial.⁩
6. ⁧**Observability מלא ב-LangSmith:** כל שאלה מייצרת עץ trace עם טוקנים, עלות ולטנציה לכל node. ככה ידענו בדיוק ש-linear עולה ⁦$0.010⁩ ו-agent ⁦$0.016⁩.⁩
7. ⁧**הכל resumable:** ה-crawl רץ לילה שלם בקצב 1 בקשה/שנייה (נימוס ל-WAF) וממשיך מאותה נקודה אחרי קריסה.⁩

## ⁧הסיפור הכי חזק למנחה: "מי שופט את השופט?"⁩

⁧אותו בוט בדיוק עבר מ-27.5% ל-91.2% correctness — בלי לגעת במודל. כל התיקון היה במדידה, בשלוש שכבות:⁩

1. ⁧**ה-gold היה שבור.** ה-CSV של Webiks נבנה לאימון embedder, אז פסקאות הזהב היו טנגנציאליות. קראנו 11 כשלים ביד, קיירטנו מחדש את כל 40 השאלות מול טקסט הדפים האמיתי → ⁦27.5%→40%⁩.⁩
2. ⁧**ההיפותזה שלנו הופרכה בניסוי.** חשבנו שסירובי-יתר באים מ-similarity floor נמוך מדי; calibration sweep הראה שה-floor לא חותך כלום — הבעיה הייתה over-refusal בשלב הגנרציה. תיקון פרומפט הוריד false refusals מ-7 ל-1, בלי לפגוע ב-100% adversarial (הוספנו guard לשאלות עצה אישית).⁩
3. ⁧**השופט עצמו נכשל בכיול.** שפטנו ביד את כל 34 התשובות מול הדפים: o4-mini הסכים עם האדם רק 56% ואפילו הפך verdict בין שתי ריצות זהות. gpt-4.1 הסכים 79% ושחזר את ספירת האדם — החלפה של שורה אחת → ⁦91.2%⁩ (האדם: 88.2%).⁩

⁧**הלקח:** LLM-as-judge הוא בעצמו מודל שחייב ולידציה. כמעט הגשנו כותרת של "73.5%" שהייתה ארטיפקט של השופט.⁩
